Imports are package-qualified (`from core.db import вҖҰ`, `from domain import vault_store`); packages resolve because entry points sit at root (`sys.path[0]`) and `tests/conftest.py`/`scripts`/`triage` insert repo root. `web_core`, `db`, `vault_store` etc. anchor `data/`/`vault/`/templates to repo root via `dirname(dirname(__file__))`.

### Conventions that bite
- **App timezone** (settings key `app_tz`, default = machine zone) for ALL "today" logic (never UTC) вҖ” always go through `db.today_iso()`/`now_sg()`/`get_tz()`, never a hardcoded zone. `now_iso()` stays UTC for audit columns. Display dates use the **`7 Jul 2025`** format (the `fdate` filter); task **due dates** use `due_label` вҖ” relative + compact (`today`/`yesterday`/`3d over`/`tomorrow`/weekday within a week/`13 Jul`, year only when not this year).
- **Voice transcription**: mlx-whisper with an EXPLICIT `language` (default `en`, settings key `voice_language`) + `condition_on_previous_text=False`, model `whisper-large-v3-mlx` (settings key `whisper_model`). Never rely on auto-detect вҖ” on short Singaporean-accented clips it mis-detected English as Malay and drifted into repetition loops. Model was bumped mediumвҶ’large-v3 because medium mis-heard short Singlish clips ("childcare for my kid" вҶ’ "toolkit for my kit") and even large-v3-turbo slipped; only full large-v3 got it right. The web note editor plays back the original recording (`/notes/<slug>/audio`) so a slipped transcription is recoverable by ear.
- **Today membership (STICKY вҖ” user-decided 2026-07-10)**: a task shows on Today iff `due==today` OR (`overdue` & not done) OR (**`planned_on<=today`** & not done вҖ” a вҳҖ plan ROLLS OVER day after day until done or unplanned; it never quietly retreats) OR completed-today (dimmed). **Nothing auto-appears** (but planned work doesn't auto-disappear either). Compare `planned_on` with `<=`, never `==` вҖ” **this applies to EVERY surface that reasons about "today"**: `today_tasks`, board pinning, the morning digest (`capture_daemon._digest_tasks`), the AI brief (`proactive.build_brief_context`), the bot's вҳҖ badges (`queries._task_line`), and all template pills; `task_plan` toggles a rolled-over plan OFF, counting a postpone.
- **Tasks board (redesigned 2026-07-10)**: columns stay `Backlog | This week | Done`, but on-today tasks (sticky rule above) **pin to the top of This week** regardless of stored `col` вҖ” `col` is untouched; on the board, today-ness is a place, not a badge. Pinned cards are excluded from drag-sort and the reorder POST (they sit on top by RULE). Cards complete via **checkbox in place** (undo toast, card moves stacks in-DOM); `/tasks/reorder` routes drags across the done boundary through `complete_task` so drag-to-Done really completes (respawn, `completed_at`) and drag-out un-completes. Week cards confess staleness in muted mono ("Nd В· KГ— moved") only past thresholds (вүҘ7 days via `tasks.week_since` or вүҘ2 postpones). **Every `col` write goes through `_WEEK_SINCE_SQL`/`set_task_col`** (routes_tasks) so the staleness clock can't drift вҖ” the bot router uses it too. **On-today вҠҶ This-week (user rule 2026-07-10)**: вҳҖ-planning a backlog task promotes it to `col='week'` (`task_plan` + router `plan_today`), and un-ticking On-today leaves it in This week at the top of the order вҖ” never back to Backlog; only an explicit drag to Backlog demotes. вҳҖ on a DONE task reopens it (+plans). Pinned cards are draggable: вҶ’Done completes, вҶ’Backlog unplans+re-homes (due-today cards snap back), within-week drags persist sort-only.
- **Capture rules** (`t:` task, `n:` note, `i:` idea, `j:` journal, bare URL вҶ’ link note, `!` вҶ’ high priority, else `#unsorted`) live in **one** function (`capture.route_capture`). **Captured/bot/refiled tasks land at the TOP of This week** (`create_task(at_top=True)`); board/editor creation stays bottom.
- **Recurrence**: `daily | weekly:<mon..sun> | monthly:<1-28>`; completing respawns a copy at next due. **Done archives after 7 days** (`archived_at`, still queryable).
- **Goals (schema v3)**: a goal is a TITLE; everything else optional. `timeframe` = week|month|quarter|year|by_date|ongoing (legacy `period`/`kind` kept, deprecated). Shape derives from fields, not `kind`: measure (`target_num`/`current_num`+`unit`) вҶ’ number bar; linked tasks вҶ’ rollup (a linked parent counts once, not subtasks); neither вҶ’ milestone with `achieved_at` toggle (POST `/goals/<id>/achieve`, bot `mark_goal_achieved`). `ongoing` never auto-archives; `by_date` archives after `end_date`. Week `period_start` = Monday.

  - **`tasks.reschedule_count`** (v4) increments when a task's `due_date` moves strictly later or a set `planned_on` is cleared (via `routes_tasks.bump_reschedule`, called from `task_plan`/`task_edit` and the router's `set_due`/`unplan`); it feeds the backlog-intelligence "postponed NГ—" signal.
