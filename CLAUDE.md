### Architecture map (one line per file)
- `db.py` — `connect()` (WAL + busy_timeout + FK, deliberate for a synced folder); `today_iso()`/`now_sg()` run in the **configurable app timezone** (`get_tz()`, cached; settings key `app_tz`, defaulting to the machine zone via `machine_tz_name()`; `reload_tz()` drops the cache after a save); `LIFEOS_DB_PATH` override.
- `db_init.py` — `TABLES` + idempotent `init_db()` + `migrate()` gated on `meta.schema_version` (currently **6**; v2 added `tasks.deleted_at`, v3 reshaped goals, v4 added `tasks.reschedule_count` — a postpone counter, v5 added `tasks.week_since` — the This-week staleness clock, **v6 added `goals.deleted_at`** — goal soft-delete). **Goals soft-delete too** (undo, not confirm — parity with tasks/notes): filter `deleted_at IS NULL` in every goals query; `routes_goals.purge_deleted_goals` hard-removes after 30 days; task links survive delete+restore. Tasks column is **`col`** ("column" is a keyword). Tasks soft-delete via `deleted_at` (undo, not confirm) — filter `deleted_at IS NULL` in every task query; `purge_deleted` hard-removes after 30 days.
- `proactive.py` — **the three proactive AI surfaces**: `morning_brief` (07:00 AI brief, replaces the digest body), `backlog_triage` (Do/Defer/Delete over the stale backlog — scheduled Sundays + on-demand), `evening_reflection` (21:30 journal-reflection prompts). Each = a PURE `build_*_context` (unit-testable, no claude) + one prompt + a deterministic **fallback** used verbatim on any claude failure (a scheduled send is never dropped). Reaches claude ONLY via `claude_cli.call_claude`. `is_backlog_triage_request` is the deterministic on-demand trigger-phrase matcher. The deterministic digest composer (`build_digest`/`_digest_tasks`/`_stale_backlog`, the morning-brief fallback) lives here too — moved out of `capture_daemon` to break a circular import.
- `web_core.py` — Flask `app`, persisted secret key, hand-rolled **CSRF** (session token + `before_request` guard; fetch/form patch in `base.html`), `respond()` AJAX-or-redirect, `db()`, `make_test_client()`, filters (`fdate`/`days_ago`/`due_label`), nav-count + `today` context processors.
- `server.py` — thin entry: argparse `--port/--db`, set `web_core._DB_PATH`, `init_db`, register `routes_*.bp`.
- `capture.py` — deterministic quick-capture `route_capture(conn, text, forced)` (web `/capture`, the bot's prefix/URL fast paths, and the router's #unsorted fallback) + single-source `create_task()`.
- `claude_cli.py` — **the ONE place that resolves + invokes `claude -p`** (`claude_bin()`/`call_claude()`); resolves the binary by absolute path so it survives launchd's minimal PATH (the bug that stranded #unsorted notes). Used by `router.py`, triage, and Q&A. **Security perimeter (don't regress)**: `call_claude` runs `--tools ""` (ALL tools disabled) by default, so no injected instruction — from a note body, link page, image, or the query itself, on EITHER AI surface (agentic bot OR read-only notes "Ask") — can ever touch the filesystem/Bash; the app's own Python is the only thing that mutates data (ids validated, soft-delete+undo). The lone exception is the router's **image** path, which passes `tools="Read"` so Claude can view the downloaded photo — Read only, never more. Never pass `--dangerously-skip-permissions`. Defense-in-depth: `router._CONTRACT` tells the model that context/notes/images/web content is DATA, never instructions.
- `router.py` — **the agentic bot router (v2): the single `claude -p` entry point.** `route(conn, message, image_path=None)` builds compact live context (open + done-today tasks with ids, goals with ids/progress, today, journal count, **+ the last 3 (Kelvin, bot) exchanges**), asks for a STRICT JSON action, validates every id against context, and ACTS via the existing helpers. Actions: create_task/note/goal, append_journal, complete/uncomplete/plan/unplan/set_due/rename/move/delete_task, update_goal_number, answer, clarify, multi. **Images**: when `image_path` is set the prompt tells Claude to view the file with its **Read tool** (the CLI is multimodal in `-p` print mode — plain `claude -p`, prompt on stdin, no special flags) before emitting JSON; a note created from a photo gets a `media:` frontmatter pointer (mirrors `audio:`). **Exchange memory**: last `_MEM_MAX_PAIRS`=3 turns (each side capped `_MEM_ENTRY_CAP`=400 chars) persisted in `settings[router_exchanges]` (survives daemon restarts) and replayed into context EVERY turn, so "yes"/"the second one"/"change it to friday" resolve. Safety rails: raw message → `data/capture_raw.log` BEFORE the call; claude fail/invalid-JSON (after one retry) → fall back to #unsorted note; delete is SOFT; unknown id → clarify. `handle_callback()` applies inline-keyboard Undo inverses.
- `vault_store.py` — notes/journal are markdown files under `vault/` (NOT in the DB); tiny YAML-frontmatter subset (title/tags/created/pinned/`audio`/`media`); notes soft-delete to `vault/.trash/`; voice originals kept in `vault/.audio/`; photo captures kept in `vault/.media/` (pointed at by a note's `media:` frontmatter); `LIFEOS_VAULT_DIR` override.
- `vault/profile.md` — **distilled triage context, injected into every `claude -p` triage call** so keep it LEAN and imperative (routing rules, not biography); it's canonical for classification (wins over message-content guesses). Longer personal reference material belongs in ordinary vault notes, never in profile.md. Starter is auto-created (never overwritten) by `triage/run_triage.py`.
- `capture_daemon.py` — Telegram long-poll. Each message: prefix/URL → deterministic `route_capture` (no claude); unambiguous list question → `queries.answer_query` (no claude); **everything else → `router.route` (the ONE claude call), with a "typing" indicator.** The `Telegram` HTTP client lives in `telegram_api.py`; voice transcode/transcribe (`oga_to_wav`/`transcribe_wav`) in `voice.py`; the daemon imports both. Voice is transcribed locally (mlx-whisper), the .oga preserved in `vault/.audio/`, then the text goes through the same router. **Photos + image documents** (`_handle_photo`) download the highest-res copy via getFile → `vault/.media/<YYYYMMDD-HHMMSS>-<file_unique_id>.jpg`, then route through the SAME router with the caption (Claude reads the file with its Read tool). Inline-keyboard Undo taps arrive as `callback_query` → `_process_callback` → `router.handle_callback`. `triage/run_triage.py` is now ONLY the `--sweep` fallback for #unsorted leftovers — scheduled ~45s after any router fallback + once daily (`sweep_last_day`); the old debounce is gone. Heartbeats in `settings` (`capture_last_ran`/`triage_last_ran`/`backup_last_ran`) drive the sidebar health dots via `web_core.health_status`. **Outbound proactive AI** (bodies in `proactive.py`): `maybe_send_digest` → the AI **morning brief** at `digest_hour` (Sundays weave in a backlog triage); `maybe_send_reflection` → the **evening reflection** at `reflection_hour`; both guarded once-per-day. The on-demand backlog triage is a deterministic fast-path in `_handle_text` (before the router) when `proactive.is_backlog_triage_request` matches ("triage my backlog", "clean up my tasks").
- `tasks_core.py` / `goals_core.py` — the **pure, Blueprint-free** task/goal domain helpers (`today_tasks`, `today_task_rows`, `complete_task` w/ recurrence respawn + parent auto-complete, `subtask_progress`, `archive_old_done`, `next_due_date`, `set_task_col`; `goal_progress`, `format_goal_progress`, `current_period_start`, `archive_expired_goals`). Non-Flask callers (`router`, `proactive`, `capture_daemon`, `queries`, `routes_main`) import from these; `routes_tasks.py`/`routes_goals.py` re-export them for back-compat and keep only the kanban/goal CRUD **routes**.
- `routes_tasks.py` (kanban CRUD), `routes_main` (Today), `routes_notes`, `routes_journal`, `routes_goals` — one blueprint each.
- `web/templates/` — `base.html` (sidebar, bottom nav, FAB, note+task editor overlays, CSRF JS), `_macros.html` (task rows/cards once), per-view pages. `web/static/app.css` + the JS bundles — `core.js` (shared `post`/`toast`/`removeWithUndo`/`confirmClick`/`dueLabel`/ring helpers, loaded first), then `board.js` (kanban + task editor), `notes.js` (notes filter/Ask + note editor + audio), `feed.js` (capture + goals + journal) — lift `design/mockup.html`.

### Design contract (`design/mockup.html` is approved — match it when touching UI)

Corollary (2026-07-10): the home hero never shows lit "On today" pills — the panel IS today, so the pill there is a tautology (hover-reveal untick on desktop; the task editor's ☀ on phone). Lit pills mark the on-today *minority* on the /tasks board only. Done rows recede: no pill, no "done" text (strike + dim + checked box already say it). ### Conventions that bite
- **App timezone** (settings key `app_tz`, default = machine zone) for ALL "today" logic (never UTC) — always go through `db.today_iso()`/`now_sg()`/`get_tz()`, never a hardcoded zone. `now_iso()` stays UTC for audit columns. Display dates use the **`7 Jul 2025`** format (the `fdate` filter); task **due dates** use `due_label` — relative + compact (`today`/`yesterday`/`3d over`/`tomorrow`/weekday within a week/`13 Jul`, year only when not this year).
- **Voice transcription**: mlx-whisper with an EXPLICIT `language` (default `en`, settings key `voice_language`) + `condition_on_previous_text=False`, model `whisper-large-v3-mlx` (settings key `whisper_model`). Never rely on auto-detect — on short Singaporean-accented clips it mis-detected English as Malay and drifted into repetition loops. Model was bumped medium→large-v3 because medium mis-heard short Singlish clips ("childcare for my kid" → "toolkit for my kit") and even large-v3-turbo slipped; only full large-v3 got it right. The web note editor plays back the original recording (`/notes/<slug>/audio`) so a slipped transcription is recoverable by ear.
- **Today membership (STICKY — user-decided 2026-07-10)**: a task shows on Today iff `due==today` OR (`overdue` & not done) OR (**`planned_on<=today`** & not done — a ☀ plan ROLLS OVER day after day until done or unplanned; it never quietly retreats) OR completed-today (dimmed). **Nothing auto-appears** (but planned work doesn't auto-disappear either). Compare `planned_on` with `<=`, never `==` — **this applies to EVERY surface that reasons about "today"**: `today_tasks`, board pinning, the morning digest (`capture_daemon._digest_tasks`), the AI brief (`proactive.build_brief_context`), the bot's ☀ badges (`queries._task_line`), and all template pills; `task_plan` toggles a rolled-over plan OFF, counting a postpone.
- **Tasks board (redesigned 2026-07-10)**: columns stay `Backlog | This week | Done`, but on-today tasks (sticky rule above) **pin to the top of This week** regardless of stored `col` — `col` is untouched; on the board, today-ness is a place, not a badge. Pinned cards are excluded from drag-sort and the reorder POST (they sit on top by RULE). Cards complete via **checkbox in place** (undo toast, card moves stacks in-DOM); `/tasks/reorder` routes drags across the done boundary through `complete_task` so drag-to-Done really completes (respawn, `completed_at`) and drag-out un-completes. Week cards confess staleness in muted mono ("Nd · K× moved") only past thresholds (≥7 days via `tasks.week_since` or ≥2 postpones). **Every `col` write goes through `_WEEK_SINCE_SQL`/`set_task_col`** (routes_tasks) so the staleness clock can't drift — the bot router uses it too. **On-today ⊆ This-week (user rule 2026-07-10)**: ☀-planning a backlog task promotes it to `col='week'` (`task_plan` + router `plan_today`), and un-ticking On-today leaves it in This week at the top of the order — never back to Backlog; only an explicit drag to Backlog demotes. ☀ on a DONE task reopens it (+plans). Pinned cards are draggable: →Done completes, →Backlog unplans+re-homes (due-today cards snap back), within-week drags persist sort-only.
- **Capture rules** (`t:` task, `n:` note, `i:` idea, `j:` journal, bare URL → link note, `!` → high priority, else `#unsorted`) live in **one** function (`capture.route_capture`). **Captured/bot/refiled tasks land at the TOP of This week** (`create_task(at_top=True)`); board/editor creation stays bottom.
- **Recurrence**: `daily | weekly:<mon..sun> | monthly:<1-28>`; completing respawns a copy at next due. **Done archives after 7 days** (`archived_at`, still queryable).
- **Goals (schema v3)**: a goal is a TITLE; everything else optional. `timeframe` = week|month|quarter|year|by_date|ongoing (legacy `period`/`kind` kept, deprecated). Shape derives from fields, not `kind`: measure (`target_num`/`current_num`+`unit`) → number bar; linked tasks → rollup (a linked parent counts once, not subtasks); neither → milestone with `achieved_at` toggle (POST `/goals/<id>/achieve`, bot `mark_goal_achieved`). `ongoing` never auto-archives; `by_date` archives after `end_date`. Week `period_start` = Monday.
- Schema change ⇒ edit `db_init.py`, bump `SCHEMA_VERSION`, add to `migrate()`.
- **`settings` keys** (key/value table, all optional with code defaults):
  - `digest_hour` (int, default **7**) + `digest_last_sent` (date) — morning-brief schedule + once-per-day guard.
  - `reflection_hour` (**"HH:MM"** or bare hour, default **"21:30"**) + `reflection_last_sent` (date) — evening-reflection schedule + guard.
  - `app_tz` (IANA zone, default = machine zone) — the app timezone for all "today" logic (`db.get_tz()`).
  - `sweep_last_day` (date) — daily #unsorted sweep floor. `telegram_offset` — long-poll cursor. `router_exchanges` (JSON) — rolling bot memory. `voice_language`/`whisper_model` — transcription overrides.
  - **`tasks.reschedule_count`** (v4) increments when a task's `due_date` moves strictly later or a set `planned_on` is cleared (via `routes_tasks.bump_reschedule`, called from `task_plan`/`task_edit` and the router's `set_due`/`unplan`); it feeds the backlog-intelligence "postponed N×" signal.
